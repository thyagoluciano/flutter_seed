Firebase Android (Login Google)

Configurar Firebase

Para criar um novo projeto Firebase, é necessário acessar este Link.

  1. Clica em Adicionar Projeto para criar um novo projeto.

  2. Insira o Nome do Projeto

  3. Habilitar ou não o uso do Google Analytics

  4. Caso tenha habilitado cadastrar uma nova conta do Google Analytics ou escolher uma já existente

  5. Aguarde a criação ser concluída e clique em Continuar

  6. Isso levará você a Página do Projeto. A partir daqui, você poderá adicionar o Firebase ao seu projeto Flutter

Configuração do Android

  1. Clique no ícone do Android

  2. Neste formulário, você deve primeiro inserir o Nome do Pacote Android. Você pode obter o nome do pacote no arquivo AndroidManifest.xml que fica dentro do diretório Android → app → src → main → AndroidManifest.xml

     <manifest xmlns:android="http://schemas.android.com/apk/res/android"
 package="br.com.seupacote.teste">
     <uses-permission android:name="android.permission.INTERNET"/>

     <application
         android:name="io.flutter.app.FlutterApplication"
         android:label="@string/app_name"
         android:icon="@mipmap/ic_launcher">

         ...
     </application>
 </manifest>

  3. Em seguida você pode inserir um Apelido do aplicativo, que é opcional.

  1. Você deve inserir o hash SHA-1 Passe o mouse sobre a ajuda (?) E clique em Ver esta página . Isso o levará à página Autenticando seu cliente mas básicamente você deve fazer os passos abaixo.

    • Gerar o arquivo app-teste-debug.keystore

        keytool -genkey -v -keystore app-teste-debug.keystore -alias app-teste-debug -keyalg RSA -keysize 2048 -validity 10000

    • Gerar o hash SHA-1

        keytool -list -v -alias app-teste-debug -keystore app-teste-debug.keystore

    • Copiar o Valor SHA1 e clicar em Registrar app

  2. A segunda etapa do processo e fazer o download do arquivo google-services.json e simplesmente arrastar e soltar no diretório do projeto Android → app → src → google-service.json

  3. Na terceira etapa, basta seguir as instruções e adicionar os trechos de código na posição desejada.

    • Alterar o arquivo Android → build.gradle

        buildscript {
      ext.kotlin_version = '1.3.50'
      repositories {
          google()  // Google's Maven repository
          jcenter()
      }

  dependencies {
      classpath 'com.android.tools.build:gradle:3.5.0'
      classpath 'com.google.gms:google-services:4.3.3' // Add this line
      classpath "org.jetbrains.kotlin:kotlin-gradle-plugin:$kotlin_version"
  }
  }

  allprojects {
      repositories {
          google()  // Google's Maven repository
          jcenter()
      }
  }
  ...

    • Alterar o arquivo Android → app → build.gradle

        ...
  apply plugin: 'com.android.application'
  apply plugin: 'com.google.gms.google-services'
  ...
      defaultConfig {
          applicationId "br.com.teste.dev"
          minSdkVersion 16
          targetSdkVersion 28
          versionCode flutterVersionCode.toInteger()
          versionName flutterVersionName
          multiDexEnabled true // Adicione esta linha
      }
  ...

  dependencies {
      // add the Firebase SDK for Google Analytics
      implementation 'com.google.firebase:firebase-analytics:17.2.2'
      implementation "org.jetbrains.kotlin:kotlin-stdlib-jdk7:$kotlin_version"
  }
  ...

  4. Por fim, para concluir esta etapa, basta clicar em Pular etapa vamos fazer mais algumas configurações para conseguir realizar o login com google.

Configurações para o Google Login

  1. Vá para Página de autenticação e clique em Configurar método de login

  2. Ná página Provedores de login temos que ativar o Google Login e prencher o nome público do projeto e o email de suporte do projeto. Em seguida clicar em Salvar

  3. Precisamos preencher configurar o consentimento do OAuth, para isso acesse este Link. Verifique no canto superior esquerdo, se seu projeto foi selecionado, escolha externo e clique em Criar

  4. Preencha o formulário de Consentimento OAuth precisamos preencher o E-mail para suporte, e os links: (Link da página inicial do aplicativo, Link da Política de Privacidade do aplicativo, Link dos Termos de Serviço do aplicativo), devemos preencher esses dados com o mesmo link exibido no Domínios autorizados que dever ser o nomeDoSeuProjeto.firebaseapp.com e clique em salvar.

  1. Por último mas não menos importante precisamos dizer para seu app utilizar o arquivo app-teste-debug.keystore configurado anteriormente.

    • Para isso vamos criar o key.properties arquivo no diretorio Android → key.properties esse arquivo tera o seguinte conteúdo.
        storePassword=suaSenha
  keyPassword=suaSenha
  keyAlias=app-teste-debug
  storeFile=/caminho/do/seu/app-teste-debug.keystore

    • Alterar o arquivo Android → app → build.gradle e adicionar a configuração do key.properties.

        ...
  def keystorePropertiesFile = rootProject.file("key.properties")
  def keystoreProperties = new Properties()
  keystoreProperties.load(new FileInputStream(keystorePropertiesFile))

  android {
      ...

      signingConfigs {
          debug {
              keyAlias keystoreProperties['keyAlias']
              keyPassword keystoreProperties['keyPassword']
              storeFile file(keystoreProperties['storeFile'])
              storePassword keystoreProperties['storePassword']
          }

          release {
              keyAlias keystoreProperties['keyAlias']
              keyPassword keystoreProperties['keyPassword']
              storeFile file(keystoreProperties['storeFile'])
              storePassword keystoreProperties['storePassword']
          }
      }

      buildTypes {
          debug {
              signingConfig signingConfigs.debug
          }

          release {
              ...
              signingConfig signingConfigs.release
              ...
          }
      }
  ...
    • Finalmente podemos testar o login com google.
